# Kotlin — язык программирования

## Что это такое?

Представь, что ты хочешь объяснить роботу, что надо делать. Чтобы робот тебя понял,
нужен специальный язык — не русский и не английский, а язык программирования.

**Kotlin** — это язык, на котором написан весь код нашего приложения. Телефон его понимает
и выполняет команды: показывает кнопки, отправляет данные, сохраняет историю.

Kotlin придумала компания JetBrains (российские разработчики!), а с 2017 года Google
объявил его основным языком для Android-разработки.

---

## Чем Kotlin лучше старых языков?

До Kotlin Android-разработчики писали на **Java** — старом и многословном языке.
Kotlin позволяет написать то же самое, но короче и безопаснее.

**На Java** (старый способ):
```java
String name = null;
int length = name.length(); // CRASH! Программа падает
```

**На Kotlin** (новый способ):
```kotlin
val name: String? = null
val length = name?.length // Всё хорошо — вернёт null, а не упадёт
```

Kotlin просто не даст тебе случайно обратиться к пустому значению — он заметит
это ещё до запуска программы.

## Ключевые особенности Kotlin в нашем проекте

### `val` и `var` — переменные

Как ящики для хранения значений:

```kotlin
val имяУстройства = "HC-05"  // val — нельзя поменять (как написать маркером)
var счётчик = 0               // var — можно поменять (как написать карандашом)
счётчик = 1                   // OK
имяУстройства = "HC-06"       // ОШИБКА — val нельзя перезаписать
```

### `fun` — функции

Функция — это кусочек кода с именем, который можно вызвать:

```kotlin
fun поздороваться(имя: String): String {
    return "Привет, $имя!"
}

val результат = поздороваться("Артём")  // "Привет, Артём!"
```

В нашем проекте так устроен, например, `TimeUtils.formatDateTime()` — принимает
числа года, месяца, дня и возвращает готовую строку "2026-04-23 14:30".

### `class` и `object` — классы и объекты

**Класс** — это чертёж (как чертёж дома).
**Объект** — это конкретный дом, построенный по чертежу.

```kotlin
// Чертёж
class Лекарство(val название: String, val время: String)

// Конкретный объект
val таблетка = Лекарство("Аспирин", "08:00")
```

В проекте так устроен `Medicine` — это чертёж одной записи в истории отправок.

### `data class` — умный класс для данных

Если добавить слово `data`, Kotlin сам научит класс сравнивать объекты и копировать их:

```kotlin
data class Medicine(val id: Long, val dateTime: String)

val запись1 = Medicine(1, "2026-04-23 14:30")
val запись2 = запись1.copy(dateTime = "2026-04-23 15:00")  // удобное копирование
```

### `object` — синглтон

Иногда нужен только один экземпляр класса на всё приложение:

```kotlin
object TimeUtils {
    fun formatDateTime(...): String { ... }
}
```

Слово `object` вместо `class` означает: «создай один экземпляр и больше не создавай».
Удобно для утилит — `TimeUtils` именно такой.

### `?` и `?.` — безопасность от пустых значений

В Kotlin тип с `?` может быть пустым (null), а без `?` — никогда:

```kotlin
val имя: String = "HC-05"    // никогда не null
val имя2: String? = null     // может быть null

val длина = имя2?.length     // если null — вернёт null, не упадёт
val длина2 = имя2 ?: "Нет имени"  // если null — подставит "Нет имени"
```

### `suspend fun` — приостанавливаемые функции

Специальные функции, которые умеют «ждать» не блокируя телефон.
Подробнее в разделе [07-корутины-и-flow.md](07-корутины-и-flow.md).

## Итого

| Концепция | Простыми словами |
|---|---|
| `val` | Значение нельзя менять — как написать маркером |
| `var` | Значение можно менять — как написать карандашом |
| `fun` | Кусочек кода с именем, который можно вызвать |
| `class` | Чертёж объекта |
| `data class` | Чертёж, который умеет сравнивать и копировать себя |
| `object` | Класс, у которого всегда только один экземпляр |
| `String?` | Строка, которая может быть пустой (null) |
| `suspend fun` | Функция, которая умеет ждать без зависания |
