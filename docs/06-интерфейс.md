# UI-компоненты — из чего собран экран

## Как строится экран в Android?

Экран в Android — как слоёный бутерброд.
Снизу — контейнер (разметка), сверху — элементы (кнопки, тексты, списки).

Всё описывается в XML-файлах в папке `res/layout/`.
Код на Kotlin «оживляет» эти элементы: вешает обработчики нажатий, меняет тексты.

---

## Layouts — контейнеры для элементов

Контейнеры решают, как расположить элементы на экране.

### LinearLayout — «стопка»

Элементы выстраиваются в ряд (горизонтально или вертикально):

```xml
<LinearLayout android:orientation="vertical">
    <TextView android:text="Первый" />
    <TextView android:text="Второй" />  ← под первым
    <TextView android:text="Третий" />  ← под вторым
</LinearLayout>
```

Используется в `activity_main.xml` и `item_medicine.xml`.

### ConstraintLayout — «свободное расположение»

Элементы можно разместить в любой точке экрана, привязав их к другим элементам
или краям экрана:

```xml
<ConstraintLayout>
    <Button
        android:id="@+id/sendButton"
        app:layout_constraintTop_toBottomOf="@id/timePicker"  ← под TimePicker
        app:layout_constraintStart_toStartOf="parent"          ← левый край экрана
        app:layout_constraintEnd_toEndOf="parent" />           ← правый край экрана
</ConstraintLayout>
```

Используется в `fragment_control.xml` и `fragment_history.xml`.

### ScrollView — прокручиваемый контейнер

Если контент не помещается на экране — ScrollView добавляет прокрутку:

```xml
<ScrollView>
    <ConstraintLayout>
        <!-- DatePicker и TimePicker большие — без прокрутки не влезут -->
    </ConstraintLayout>
</ScrollView>
```

Используется в `fragment_control.xml` — DatePicker и TimePicker высокие.

## Основные элементы интерфейса

### TextView — текстовая метка

Просто отображает текст. Не нажимается.

```xml
<TextView
    android:id="@+id/connectionStatusText"
    android:text="Статус: Не подключено"
    android:textSize="14sp"
    android:textColor="@android:color/holo_red_dark" />
```

В `ControlFragment` цвет меняется из кода:
```kotlin
connectionStatusText.setTextColor(ContextCompat.getColor(..., android.R.color.holo_green_dark))
```

### Button — кнопка

Нажимается. Обработчик задаётся в коде:

```kotlin
sendButton.setOnClickListener {
    // что делать при нажатии
    viewModel.sendMedicineTime(...)
}
```

### DatePicker — выбор даты

Виджет для выбора года, месяца и дня. В нашем приложении используется режим `spinner`
(барабаны с прокруткой, не календарь):

```xml
<DatePicker
    android:datePickerMode="spinner"
    android:calendarViewShown="false" />  ← скрываем календарь, оставляем барабаны
```

Из кода читаем выбранное значение:
```kotlin
val year  = datePicker.year
val month = datePicker.month    // 0 = январь, 11 = декабрь!
val day   = datePicker.dayOfMonth
```

### TimePicker — выбор времени

Виджет для выбора часов и минут. Тоже используем режим `spinner`:

```xml
<TimePicker android:timePickerMode="spinner" />
```

```kotlin
val hour   = timePicker.hour    // 0–23
val minute = timePicker.minute  // 0–59
```

## ViewPager2 и TabLayout — вкладки

### Как это работает

Представь записную книжку с закладками — это и есть вкладки.
Нажимаешь на закладку «Управление» — видишь один экран.
Нажимаешь «История» — видишь другой. Можно переключаться свайпом.

```
┌─────────────────────────────┐
│ [Управление] │ [История]    │  ← TabLayout
├─────────────────────────────┤
│                             │
│   (содержимое вкладки)      │  ← ViewPager2
│                             │
└─────────────────────────────┘
```

### ViewPager2

**ViewPager2** — контейнер, в котором живут фрагменты-вкладки.
Поддерживает свайп между вкладками:

```xml
<androidx.viewpager2.widget.ViewPager2
    android:id="@+id/viewPager"
    android:layout_width="match_parent"
    android:layout_height="0dp"
    android:layout_weight="1" />
```

### TabLayout

**TabLayout** — полоска с кнопками вкладок вверху:

```xml
<com.google.android.material.tabs.TabLayout
    android:id="@+id/tabLayout"
    app:tabMode="fixed"
    app:tabGravity="fill" />  ← вкладки растягиваются на всю ширину
```

### Связка через TabLayoutMediator

`TabLayoutMediator` — «клей» между TabLayout и ViewPager2:

```kotlin
TabLayoutMediator(tabLayout, viewPager) { tab, position ->
    tab.text = adapter.getPageTitle(position)  // "Управление" или "История"
}.attach()
```

### ViewPagerAdapter — адаптер вкладок

Говорит ViewPager2, какой фрагмент показывать на каждой позиции:

```kotlin
class ViewPagerAdapter(activity: FragmentActivity) : FragmentStateAdapter(activity) {
    override fun getItemCount() = 2

    override fun createFragment(position: Int): Fragment {
        return when (position) {
            0 -> ControlFragment()
            1 -> HistoryFragment()
            else -> throw IllegalStateException()
        }
    }
}
```

## RecyclerView — умный прокручиваемый список

**RecyclerView** — компонент для отображения длинных списков.

Называется «Recycler» (переработчик), потому что он **повторно использует** элементы:
когда карточка уходит за верхний край экрана, RecyclerView берёт её и
подставляет снизу с новыми данными. Это экономит память.

Для работы RecyclerView нужны три части:

### 1. Layout элемента — `item_medicine.xml`

Как выглядит одна карточка в списке:

```xml
<CardView app:cardElevation="4dp">  ← тень под карточкой
    <LinearLayout>
        <TextView android:id="@+id/dateTimeText" />   ← дата приёма
        <TextView android:id="@+id/deviceNameText" />  ← имя устройства
        <TextView android:id="@+id/sentAtText" />      ← когда отправили
        <Button android:id="@+id/editButton" />
        <Button android:id="@+id/deleteButton" />
    </LinearLayout>
</CardView>
```

### 2. Adapter — `MedicineAdapter`

Адаптер соединяет список данных и визуальные карточки:

```kotlin
class MedicineAdapter(...) : ListAdapter<Medicine, MedicineViewHolder>(...) {

    // Создать новую карточку (когда их ещё не хватает)
    override fun onCreateViewHolder(...) = MedicineViewHolder(...)

    // Заполнить карточку данными
    override fun onBindViewHolder(holder, position) {
        holder.bind(getItem(position))
    }
}
```

### 3. DiffUtil — умное обновление списка

Когда список меняется, DiffUtil сравнивает старый и новый список
и обновляет только те карточки, которые изменились.

Без DiffUtil — перерисовывается весь список (медленно и мигает).
С DiffUtil — перерисовывается только изменённая карточка (быстро и плавно).

```kotlin
class MedicineDiffCallback : DiffUtil.ItemCallback<Medicine>() {
    // Это одна и та же запись?
    override fun areItemsTheSame(old: Medicine, new: Medicine) = old.id == new.id

    // Содержимое записи изменилось?
    override fun areContentsTheSame(old: Medicine, new: Medicine) = old == new
}
```

## CardView — карточка с тенью

**CardView** — контейнер, который выглядит как белая карточка с тенью и скруглёнными углами.

```xml
<androidx.cardview.widget.CardView
    app:cardCornerRadius="8dp"   ← скруглённые углы
    app:cardElevation="4dp">     ← высота тени
    <!-- содержимое карточки -->
</androidx.cardview.widget.CardView>
```

## Material Design 3 — дизайн-система Google

**Material Design** — это набор правил и готовых компонентов от Google для создания
красивых и удобных приложений.

В проекте используется **Material Design 3** (версия 1.13.0).
Например, кнопки «Редактировать» и «Удалить» — это `OutlinedButton` из Material:

```xml
<Button
    style="@style/Widget.Material3.Button.OutlinedButton"
    android:text="Редактировать" />
```

`OutlinedButton` — кнопка с рамкой, без заливки. Выглядит аккуратнее обычной.

## AlertDialog — всплывающий диалог

**AlertDialog** — стандартный диалог подтверждения действия.

```kotlin
AlertDialog.Builder(requireContext())
    .setTitle("Удалить запись?")
    .setMessage("Вы уверены?")
    .setPositiveButton("Удалить") { _, _ ->
        viewModel.deleteMedicine(medicine)
    }
    .setNegativeButton("Отмена", null)
    .show()
```

Используется при удалении записи и очистке всей истории.

---

## Toast — короткое уведомление

**Toast** — маленькое всплывающее сообщение внизу экрана.
Появляется на пару секунд и исчезает само:

```kotlin
Toast.makeText(this, "Данные отправлены!", Toast.LENGTH_SHORT).show()
```
