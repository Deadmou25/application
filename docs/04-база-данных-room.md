# Room — как приложение запоминает историю

## Зачем нужна база данных?

Когда приложение закрыто, все данные в памяти телефона исчезают.
Чтобы история отправок сохранялась между запусками — нужна **база данных**.

База данных — это как тетрадка, в которую приложение записывает данные.
Тетрадка лежит на телефоне и не пропадает при перезапуске.

---

## SQLite — встроенная «тетрадка» в каждом телефоне

**SQLite** — лёгкая база данных, которая встроена в каждый Android-телефон.
Данные хранятся в виде таблиц — как Excel.

Таблица `medicines` в нашем проекте выглядит так:

| id | dateTime | sentAt | deviceName |
|---|---|---|---|
| 1 | 2026-04-23 14:30 | 1745408400000 | HC-05 |
| 2 | 2026-04-23 18:00 | 1745423200000 | HC-05 |
| 3 | 2026-04-24 08:00 | 1745488600000 | null |

Каждая строка = одна запись о том, что данные были отправлены на Arduino.

---

## Room — умная обёртка над SQLite

Писать SQL-запросы вручную скучно и сложно. **Room** — это библиотека от Google,
которая позволяет работать с базой данных на Kotlin, без написания сырого SQL
(хотя SQL-запросы всё равно можно писать — Room их выполняет под капотом).

Room состоит из трёх частей — как три слоя торта:

```
┌─────────────────────────┐
│   Entity (Сущность)     │  ← описывает одну строку таблицы
├─────────────────────────┤
│   DAO (Интерфейс)       │  ← методы для работы с таблицей
├─────────────────────────┤
│   Database (База)       │  ← точка входа, создаёт соединение
└─────────────────────────┘
```

---

## Entity — описание строки таблицы

**Entity** (`Medicine.kt`) — это класс, каждое поле которого становится столбцом в таблице.

```kotlin
@Entity(tableName = "medicines")  // имя таблицы в БД
data class Medicine(
    @PrimaryKey(autoGenerate = true)  // уникальный номер, создаётся автоматически
    val id: Long = 0,
    val dateTime: String,   // "2026-04-23 14:30"
    val sentAt: Long,       // число миллисекунд с 1 января 1970
    val deviceName: String? // имя устройства, может быть пустым
)
```

Аннотации (слова со `@`) — это подсказки для Room:
- `@Entity` — «это класс-таблица»
- `@PrimaryKey` — «это главный столбец, по нему ищем строки»
- `autoGenerate = true` — «сам придумай номер для каждой новой записи»

## DAO — интерфейс для работы с таблицей

**DAO** (Data Access Object — Объект Доступа к Данным) — это интерфейс,
в котором описаны все операции с таблицей.

Думай о нём как о пульте управления телевизором:
кнопка «вкл», «выкл», «громкость +» — всё готово, жми и не думай о схемах внутри.

```kotlin
@Dao
interface MedicineDao {

    // Получить все записи (самые новые сверху)
    @Query("SELECT * FROM medicines ORDER BY sentAt DESC")
    fun getAll(): Flow<List<Medicine>>

    // Добавить новую запись
    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insert(medicine: Medicine): Long

    // Изменить запись
    @Update
    suspend fun update(medicine: Medicine)

    // Удалить одну запись
    @Delete
    suspend fun delete(medicine: Medicine)

    // Удалить все записи
    @Query("DELETE FROM medicines")
    suspend fun deleteAll()
}
```

Каждый метод — одна операция. Room сам превращает их в SQL-запросы.

## AppDatabase — «замок» на базе данных

**AppDatabase** — это точка входа в базу данных. Через неё получают DAO.

Важно: база данных должна быть одна на всё приложение (синглтон).
Если создать два подключения — данные могут повредиться.

```kotlin
@Database(entities = [Medicine::class], version = 1)
abstract class AppDatabase : RoomDatabase() {

    abstract fun medicineDao(): MedicineDao  // даём доступ к DAO

    companion object {
        @Volatile  // все потоки видят одно и то же значение
        private var INSTANCE: AppDatabase? = null

        fun getDatabase(context: Context): AppDatabase {
            // Если уже создана — возвращаем ту же
            // Если нет — создаём первый и единственный раз
            return INSTANCE ?: synchronized(this) {
                Room.databaseBuilder(context, AppDatabase::class.java, "medicine_database")
                    .build()
                    .also { INSTANCE = it }
            }
        }
    }
}
```

Это паттерн **Singleton** (Одиночка) — как у жителей деревни один колодец.

## Flow — живые данные из базы

Обычный запрос к базе возвращает данные один раз. Но `getAll()` возвращает `Flow` —
это значит, что когда данные в базе изменятся, экран обновится **автоматически**.

```kotlin
// В DAO:
fun getAll(): Flow<List<Medicine>>

// В ViewModel:
val medicines: Flow<List<Medicine>> = repository.getAllMedicines()

// В HistoryFragment:
viewModel.medicines.collectLatest { список ->
    adapter.submitList(список)  // список на экране обновляется сам!
}
```

Как RSS-лента: подписался — и новые статьи появляются сами, не нужно обновлять вручную.

## KAPT — как Room читает аннотации

Room использует **аннотации** (`@Entity`, `@Dao`, `@Query` и т.д.), чтобы понять,
как устроена база данных. Эти аннотации обрабатывает специальный инструмент — **KAPT**
(Kotlin Annotation Processing Tool).

Во время сборки проекта KAPT читает все `@`-аннотации и генерирует Kotlin-код,
который Room будет выполнять. Это происходит автоматически при компиляции.

В `build.gradle.kts` это выглядит так:
```kotlin
plugins {
    id("kotlin-kapt")  // подключаем обработчик аннотаций
}
dependencies {
    kapt(libs.androidx.room.compiler)  // генератор кода для Room
}
```

## Итого

| Класс | Аннотация | Роль |
|---|---|---|
| `Medicine` | `@Entity` | Одна строка в таблице БД |
| `MedicineDao` | `@Dao` | Методы для чтения/записи/удаления |
| `AppDatabase` | `@Database` | Главная точка входа, синглтон |
| Поле `id` | `@PrimaryKey` | Уникальный идентификатор строки |

| Операция | Аннотация |
|---|---|
| Получить данные | `@Query("SELECT ...")` |
| Добавить | `@Insert` |
| Изменить | `@Update` |
| Удалить | `@Delete` |
