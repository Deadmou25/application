# Техническое задание: Передача времени приёма лекарств с Android-устройства на Arduino по Bluetooth

## Инструменты разработки (Android)

Для разработки Android-приложения рекомендуется использовать следующие инструменты:

- **IDE**: Android Studio (последняя стабильная версия, например, Android Studio Iguana | 2023.2.1+)
- **Язык программирования**: Kotlin (рекомендуется) или Java
- **Build system**: Gradle (встроен в Android Studio)
- **Min SDK**: API 21 (Android 5.0 Lollipop) — для поддержки большинства устройств с Bluetooth
- **Target SDK**: актуальная версия (например, API 34 — Android 14)
- **Эмулятор / Устройство**:
    - Физическое Android-устройство с поддержкой Bluetooth (эмулятор Android Studio **не поддерживает** Bluetooth-соединение с внешними устройствами)
- **Версия JDK**: JDK 17 (встроена в Android Studio начиная с версии Arctic Fox)

> **Примечание**: Для отладки Bluetooth-взаимодействия обязательно требуется реальное устройство.

## 2.1. Требования к Android-приложению

### Основные элементы интерфейса

- **Поле выбора даты** — `DatePicker`.
- **Поле выбора времени** — `TimePicker`.
- **Кнопка «ОК»** — для отправки выбранных даты и времени на Arduino.

### Управление Bluetooth-соединением

- Поиск доступных Bluetooth-устройств.
- Подключение к выбранному устройству (Arduino с модулем HC-05 или HC-06).
- Отображение статуса соединения:
    - *Подключено* / *Не подключено*.

### Логика отправки данных

При нажатии кнопки **«ОК»**:

1. Приложение формирует строку в формате: `YYYY-MM-DD HH:MM`  
   Пример: `2026-01-13 14:30`
2. Строка дополняется символом новой строки (`\n`) и отправляется по Bluetooth на подключённое устройство (Arduino).

### Обработка ошибок

- Если **Bluetooth выключен** — показать предупреждение.
- Если **нет активного подключения к Arduino** — показать предупреждение.
- Если **отправка данных не удалась** — показать уведомление об ошибке.

### Дополнительные (опциональные) функции

- Хранение истории отправленных записей локально (например, с использованием Room или SQLite).
- Возможность редактирования и удаления записей из истории.
- Автоматическая отправка данных при каждом изменении даты/времени (без кнопки «ОК»).
- Шифрование или контрольная сумма — **не требуется** на данном этапе.

## 2.2. Требования к Arduino

### Аппаратное обеспечение

- Arduino должна быть оснащена Bluetooth-модулем (например, HC-05 или HC-06).

### Обработка входящих данных

При получении строки от Android:

1. Выполняется упрощённая проверка корректности формата:
    - Наличие пробела между датой и временем.
    - Наличие двоеточия во времени.
    - Общая длина строки ≈ 16 символов + символ новой строки (`\n`).

2. Если данные **корректны**:
    - Включается/мигает светодиод (например, 3 раза с интервалом 200 мс) — визуальное подтверждение приёма.
    - Данные могут:
        - Сохраняться во внутреннюю память (EEPROM).
        - Выводиться в Serial Monitor (для отладки).

3. Если данные **некорректны**:
    - Игнорируются.
    - (Опционально) мигнуть один раз красным цветом (если используется RGB-светодиод).

> **Примечание**: Передача данных — **однонаправленная** (Android → Arduino).  
> Arduino **не отправляет** подтверждение обратно.

## 4. Формат передаваемых данных

- **Тип**: текстовая строка в кодировке UTF-8.
- **Формат**: `YYYY-MM-DD HH:MM`
- **Пример**: `2026-01-13 14:30`
- **Завершение**: строка обязательно заканчивается символом новой строки `\n`.

> На стороне Arduino чтение можно реализовать с помощью:
> ```cpp
> String received = bluetoothSerial.readStringUntil('\n');
> ```
